## 🚀 快速开始

### 查看前端预览

```bash
# 进入前端预览目录
cd ERP_前端预览

# 方式1：使用 Python 简单服务器
python -m http.server 3000

# 方式2：使用 Node.js http-server
npx http-server -p 3000

# 然后浏览器打开 http://localhost:3000
```

### 编辑需求文档

直接编辑 `ERP_需求文档/` 目录下的 `.md` 文件，按照文档中的注释和示例填充内容。

## 📝 文档规范

### 需求文档结构

每个模块的需求文档包含以下章节：

1. **模块概述** - 模块定位、核心职责
2. **页面结构**
   - 页签定义
   - 每个页签的详细设计：列表字段、查询条件、操作列、批量操作、下钻页面、跨模块跳转
3. **业务逻辑**
   - 数据来源
   - 状态定义与流转
   - 生命周期
   - 跨模块联动
4. **表单设计** - 新建/编辑表单字段
5. **数据模型** - 实体及字段定义（供后端参考）

### 前端预览规范

- 使用纯 HTML + CSS + JS，不依赖构建工具
- 页面切换通过简单的 JS 函数实现
- 样式参考 Ant Design 设计规范
- 只展示效果，不做真实数据交互

## 🤖 AI 开发指南

本项目使用 AI 辅助开发，AI 需遵守以下约束：

1. **先读文档再开发** - 任何代码修改前必须先阅读对应模块的 MD 需求文档
2. **保持结构一致** - 新增模块需遵循现有文档结构和前端页面结构
3. **代码简洁** - 单个函数复杂度不超过 20，做好函数解耦
4. **中文优先** - 所有输出使用中文，包括注释和文档
5. **验证后再提交** - 修改后需在本地验证效果

详细约束见 [CLAUDE.md](./CLAUDE.md)
