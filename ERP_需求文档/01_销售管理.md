# 销售管理模块

## 1. 模块概述
销售管理模块是 ERP 系统的核心业务模块之一，负责管理从客户下单到订单履约完成的全流程。主要服务对象为销售人员和销售管理人员。

---

### 2.1 页签定义

| 页签名称 | 用途说明 |
|---------|---------|
| 销售订单 | 管理所有销售订单，支持新建、编辑、查看、取消等操作 |
| 销售退货 | 管理所有销售退货单，支持新建、查看等操作 |

| 字段名 | 类型 | 说明 | 宽度建议 |

| 规格 | 文本 | 商品规格描述 | 80px |
| 包装单位 | 文本 | 商品包装单位 | 80px |

- 订单总金额 = 所有商品小计之和 - 退货商品的成交金额

- 商品退货后小计 = 含税单价 × (原数量 - 退货数量)

#### 2.2.3.1 发货功能说明

**触发条件：**
- 订单状态为已审核
- 订单类型为直运订单
- 关联的采购单状态为已采购

**发货操作流程：**
1. 点击"发货"按钮，弹出发货弹窗
2. 展示该订单所有商品明细，默认全部勾选
3. 支持取消勾选部分商品
4. 输入快递单号，系统自动根据前缀识别快递公司：
   - SF-顺丰
   - JD-京东
   - DP-德邦
   - YZ-邮政
   - YT-圆通
5. 如前缀不符合上述识别码，支持手动选择快递公司
6. 展示格式：快递公司;快递单号

#### 2.2.3.2 合同模板维护

**入口位置：** 系统设置 → 合同模板管理

**功能说明：**
- 支持新增、编辑、删除合同模板
- 模板支持变量替换：{订单编号}、{客户名称}、{订单金额}、{商品明细}、{收件信息}、{日期}等

## 3. 业务逻辑

### 3.1 数据来源

| 数据 | 来源方式 | 来源说明 |
|------|---------|---------|
| 销售订单 | 用户手动创建 | 销售人员在列表页点击"新建订单"按钮创建 |

### 3.2 状态定义

#### 3.2.1 订单状态

| 状态 | 含义 | 可执行操作 |
|------|------|-----------|
| 草稿 | 新建未提交 | 重编、删除、提交 |
| 已审核 | 审核通过，终点状态 | 详情、重编、合同、发货、加单 |

#### 3.2.2 下推状态

| 状态 | 含义 | 说明 |
|------|------|------|
| 全部下推 | 该订单中有直运需要采购的单子已经全部下推 | 直运商品全部下推完成 |
| 部分下推 | 该订单中有直运需要采购的单子已经部分下推 | 直运商品部分下推完成 |
| 无需下推 | 有库存 | 商品无需采购 |

> **备注**：已经发货的产品，不可以再次下推采购。

#### 3.2.3 发货状态

| 状态 | 含义 | 说明 |
|------|------|------|
| 未发货 | 未进行任何发货操作 | 出库单未审核或未创建 |
| 部分发货 | 部分商品已发货 | 出库单部分审核 |
| 全部发货 | 全部商品已发货 | 当该订单除去退货的产品金额其他都已发货，该订单状态也为全部发货 |
| 无需发货 | 不需要发货操作 | 该订单有退货，且退货金额和订单金额一致 |

#### 3.2.4 开票状态（发票状态）

| 状态 | 含义 | 说明 |
|------|------|------|
| 未开票 | 未开具发票 | 应收单未开票 |
| 全部开票 | 全部金额已开票 | 当该订单除去退货的产品金额其他都已开票，该订单状态也为全部开票 |
| 部分开票 | 部分金额已开票 | 部分金额开票 |
| 无需开票 | 不需要开票 | 满足以下条件之一：<br>- 该订单有退货且退货金额和销售订单金额一致<br>- 该订单金额为0<br>- 该订单开票类型为现金 |

#### 3.2.5 财务状态

| 状态 | 含义 | 说明 |
|------|------|------|
| 未付款 | 未收到任何款项 | 应收单未收款 |
| 部分付款 | 收到部分款项 | 部分金额核销 |
| 全部付款 | 全部款项已收到 | 当该订单除去退货的产品金额其他都已付款，该订单状态也为全部付款 |
| 无需付款 | 不需要付款 | 满足以下条件之一：<br>- 该订单有退货且退货金额和销售订单金额一致，并且财务状态为未收款<br>- 该订单金额为0 |

### 3.3 状态流转

```
草稿 → 已审核：用户点击"提交"，订单自动审核通过
```

### 3.4 生命周期

1. **创建**：销售人员点击"新建订单"，填写客户、商品、收货信息
2. **保存**：点击"保存"，状态=草稿
3. **保存并提交**：点击"保存并提交"，状态=已审核
4. **执行**：已审核订单可生成出库单、应收单

### 3.5 跨模块联动

| 本模块操作 | 联动模块 | 联动效果 |
|-----------|---------|---------|
| 销售订单审核通过 | 库存管理 | 可生成出库单，锁定对应库存 |
| 销售订单审核通过 | 财务管理 | 可生成应收单，记入应收账款 |
| 销售订单取消 | 库存管理 | 释放已锁定的库存 |
| 销售订单取消 | 财务管理 | 作废对应的应收单（如已生成） |

### 3.6 权限控制

#### 3.6.1 普通用户权限

| 功能模块 | 权限说明 |
|---------|---------|
| 销售订单 | 查看归属于自己名下的客户订单 |
| 销售订单 | 创建订单 |
| 销售订单 | 编辑订单 |
| 销售订单 | 删除订单 |
| 销售订单 | 审核订单（提交即审核） |
| 发货管理 | 查看和操作发货 |
| 退货管理 | 查看和操作退货 |

**说明：**
- 普通用户只能查看和操作归属于自己名下的客户订单
- 普通用户可以创建、编辑、删除、审核订单
- 普通用户可以进行发货管理和退货管理操作

#### 3.6.2 管理员权限

| 功能模块 | 权限说明 |
|---------|---------|
| 销售订单 | 查看所有订单（包括所有业务员的订单） |
| 销售订单 | 创建/编辑/删除/审核订单 |
| 发货管理 | 全部权限 |
| 退货管理 | 全部权限 |
| 订单统计 | 查看业务统计报表 |

## 4. 表单设计

### 4.1 新建/编辑销售订单表单

表单分为四个区域：**基础信息**、**商品明细**、**订单汇总**、**备注信息**

#### 4.1.1 基础信息

| 字段名 | 类型 | 必填 | 默认值 | 校验规则 | 说明 |
|-------|------|------|-------|---------|------|
| 订单日期 | 日期 | 是 | 当前日期 | 不能为空 | 订单创建日期 |
| 期望交货日期 | 日期 | 否 | 空 | - | 期望交货的日期 |
| 客户 | 下拉搜索选择 | 是 | 无 | 必须选择有效客户 | 支持模糊查询、客户编码查询；选择后自动带出收货信息、开票信息、结算方式 |
| 结算方式 | 下拉选择 | 是 | 自动带出 | 不能为空 | 选择客户后自动填充，可手动修改 |
| 第三方平台 | 下拉选择 | 否 | 空 | - | 可选项：博赛乐-锐竞采购平台、博赛乐-虫洞空间、睿沃特-供应室、兹罗-喀斯玛、智达康-库巴扎、兹罗-库巴扎 |
| 平台订单号 | 文本 | 否 | 空 | - | 第三方平台的订单编号 |
| 收货信息 | 下拉选择 | 是 | 自动带出 | 不能为空 | 选择客户后自动填充默认地址；支持快速新建；支持智能识别粘贴的地址信息 |
| 开票信息 | 下拉选择 | 是 | 自动带出 | 不能为空 | 选择客户后自动填充默认开票信息 |

**客户选择功能说明：**
- 支持模糊查询客户名称
- 支持客户编码查询
- 选择客户后自动带出：收货信息、开票信息、结算方式

**收货信息智能识别：**
- 支持粘贴一整段地址信息
- 自动识别收货人、收货电话、收货地址

#### 4.1.2 商品明细（子表格，可动态添加行）

| 字段名 | 类型 | 必填 | 默认值 | 校验规则 | 说明 |
|-------|------|------|-------|---------|------|
| 品牌 | 下拉/文本 | 否 | 自动带出 | - | 规格编号选择后自动填充；当同一规格存在多个品牌时，自动切换为下拉选择 |
| 规格编号 | 文本 | 否 | 自动带出 | 支持模糊查询 | 输入时支持下拉列表模糊匹配；选择规格后自动带出品牌和产品信息 |
| 产品名称 | 下拉搜索选择 | 是 | 无 | 必须选择有效商品 | 选择后自动带出相关信息 |
| 包装 | 文本 | 否 | 自动带出 | - | 选择商品后自动填充 |
| 备注 | 文本 | 否 | 空 | - | 商品备注 |
| 原价 | 金额 | 是 | 自动带出 | 大于0 | 可手动修改 |
| 折扣 | 数字 | 是 | 自动带出 | 0-1之间 | 根据客户和税率默认带出，可手动修改 |
| 数量 | 数量 | 是 | 1 | 大于0的整数 | 手动输入 |
| 税率 | 数字 | 是 | 自动带出 | 大于等于0 | 根据客户默认带出，可手动修改 |
| 含税价格 | 金额 | 是 | 自动计算 | 大于0 | 原价 × 折扣 × (1 + 税率)，不可编辑 |
| 合计 | 金额 | 是 | 自动计算 | - | 含税价格 × 数量，不可编辑 |
| 库存 | 数字 | 否 | 自动带出 | - | 展示该商品的库存情况 |
| 发货方式 | 下拉选择 | 是 | 直运 | - | 可选项：直运、物流、自提、送货 |
| 指定仓库 | 下拉选择 | 否 | 空 | 非直运时必填 | 非直运方式时必须选择 |

**操作：**
- 支持添加行、删除行
- 支持同一个产品添加多次
- 支持批量修改发货方式和仓库

**字段联动计算：**
- 含税价格 = 原价 × 折扣
- 合计 = 含税价格 × 数量
- 税率仅用于后续开票参考，不参与价格计算

#### 4.1.3 订单汇总

| 字段名 | 类型 | 说明 |
|-------|------|------|
| 产品总数量 | 数字 | 所有商品数量之和 |
| 商品折后价 | 金额 | 所有商品合计之和 |
| 运费 | 金额 | 根据品牌自动填写，支持手动修改 |
| 总金额 | 金额 | 商品折后价 + 运费 |

**运费规则：**
- 根据订单中商品的品牌自动匹配设置的品牌运费
- 支持手动修改运费金额
- 加单功能时运费默认为0

#### 4.1.4 备注信息

| 字段名 | 类型 | 必填 | 默认值 | 校验规则 | 说明 |
|-------|------|------|-------|---------|------|
| 备注 | 文本（多行） | 否 | 空 | - | 订单备注信息 |

#### 4.1.5 发货方式校验规则

| 发货方式 | 是否需要选择仓库 | 说明 |

## 3. 业务逻辑（新增）

### 3.2 退货单状态定义

| 状态 | 含义 | 可执行操作 |
|------|------|-----------|
| 待审核 | 退货单已提交，等待审核 | 审核通过、审核驳回 |
| 已完成 | 退货单审核通过 | 仅查看 |

### 3.3 退货业务流程

1. **选择销售订单**：新建退货单时选择销售订单，自动带出该订单所有产品明细（包括运费）
2. **填写退货信息**：选择要退货的产品，填写退货数量
3. **选择退货方式**：根据销售订单的发货状态和财务状态选择可用的退货方式
4. **保存并提交**：提交退货单
5. **审核通过**：审核通过后执行以下操作：
   - 生成退货单
   - 关联销售订单金额自动扣减退货产品的成交金额
   - 关联销售订单成本自动扣减退货产品的成本金额
   - 如销售订单有仓库发货，在入库管理生成"退货待入库"记录
   - 关联销售订单的发货状态增加"有退货"提示
   - 关联销售订单明细的"退/换/补货数量"更新显示退货数量（退货方式，金额）

### 3.4 退货单审核通过后的联动效果

| 联动模块 | 效果 |
|---------|------|
| 销售订单 | 订单金额 -= 退货金额，订单成本 -= 退货产品成本，发货状态增加"有退货"标识 |
| 销售订单明细 | "退/换/补货数量"列显示：退货数量（退货方式，金额），return_quantity 字段更新 |
| 库存管理 | 生成"退货待入库"记录，操作入库后自动更新库存 |
| 财务管理 | 根据退货方式进行相应账务处理（冲抵、转预存、退回账户） |

### 3.5 发货状态更新

| 状态 | 说明 |
|------|------|
| 未发货（有退货） | 未发货但有退货 |
| 部分发货（有退货） | 部分发货且有退货 |
| 全部发货（有退货） | 全部发货且有退货 |
