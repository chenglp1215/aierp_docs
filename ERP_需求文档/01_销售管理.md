# 销售管理模块

## 1. 模块概述
销售管理模块是 ERP 系统的核心业务模块之一，负责管理从客户下单到订单履约完成的全流程。主要服务对象为销售人员和销售管理人员。

## 3. 业务逻辑

### 3.1 数据来源

| 数据 | 来源方式 | 来源说明 |
|------|---------|---------|
| 销售订单 | 用户手动创建 | 销售人员在列表页点击"新建订单"按钮创建 |

### 3.2 状态定义

| 状态 | 含义 | 可执行操作 |
|------|------|-----------|
| 草稿 | 新建未提交 | 编辑、提交审核、取消 |
| 待审核 | 已提交等待审批 | 审批通过、审批驳回、取消 |
| 已审核 | 审批通过 | 生成出库单、生成应收单 |
| 已完成 | 全部出库且已收款 | 仅查看 |
| 已取消 | 订单作废 | 仅查看 |

### 3.3 状态流转

```
草稿 → 待审核：用户点击"提交审核"
待审核 → 已审核：审批人点击"通过"
待审核 → 草稿：审批人点击"驳回"（驳回后可重新编辑再提交）
已审核 → 已完成：全部明细行出库完毕且应收核销完成
草稿/待审核 → 已取消：用户点击"取消"
```

### 3.4 生命周期

1. **创建**：销售人员点击"新建订单"，填写客户、商品、收货信息，状态=草稿
2. **提交**：确认无误后点击"提交审核"，状态=待审核
3. **审批**：审批人审核通过/驳回
4. **执行**：审核通过后可生成出库单、应收单
5. **完成**：出库完毕且收款完成，状态=已完成

### 3.5 跨模块联动

| 本模块操作 | 联动模块 | 联动效果 |
|-----------|---------|---------|
| 销售订单审核通过 | 库存管理 | 可生成出库单，锁定对应库存 |
| 销售订单审核通过 | 财务管理 | 可生成应收单，记入应收账款 |
| 销售订单取消 | 库存管理 | 释放已锁定的库存 |
| 销售订单取消 | 财务管理 | 作废对应的应收单（如已生成） |

## 4. 表单设计

### 4.1 新建/编辑销售订单表单

表单分为三个区域：**客户信息**、**商品明细**、**收货信息**

#### 4.1.1 客户信息

| 字段名 | 类型 | 必填 | 默认值 | 校验规则 | 说明 |
|-------|------|------|-------|---------|------|
| 客户 | 下拉搜索选择 | 是 | 无 | 必须选择有效客户 | 选择客户后自动带出联系人、电话 |
| 联系人 | 文本 | 否 | 自动带出 | - | 选择客户后自动填充，可手动修改 |
| 联系电话 | 文本 | 否 | 自动带出 | 手机号/座机格式 | 选择客户后自动填充，可手动修改 |

#### 4.1.2 商品明细（子表格，可动态添加行）

| 字段名 | 类型 | 必填 | 默认值 | 校验规则 | 说明 |
|-------|------|------|-------|---------|------|
| 商品 | 下拉搜索选择 | 是 | 无 | 必须选择有效商品 | 选择后自动带出单价、单位 |
| 单价 | 金额 | 是 | 自动带出 | 大于0 | 可手动修改 |
| 数量 | 数量 | 是 | 1 | 大于0的整数 | 手动输入 |
| 单位 | 文本 | 是 | 自动带出 | - | 选择商品后自动填充 |
| 小计 | 金额 | 是 | 自动计算 | - | 单价 × 数量，不可编辑 |

操作：支持添加行、删除行

#### 4.1.3 收货信息

| 字段名 | 类型 | 必填 | 默认值 | 校验规则 | 说明 |
|-------|------|------|-------|---------|------|
| 收货人 | 文本 | 是 | 同联系人 | 不能为空 | 默认填充客户联系人，可修改 |
| 收货电话 | 文本 | 是 | 同联系电话 | 手机号/座机格式 | 默认填充客户电话，可修改 |
| 收货地址 | 文本（多行） | 是 | 空 | 不能为空 | 手动输入详细收货地址 |
| 备注 | 文本（多行） | 否 | 空 | - | 订单备注信息 |

## 5. 数据模型

### 5.1 实体：销售订单（sale_order）

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| id | bigint | 是 | 主键 |
| order_no | varchar(32) | 是 | 订单编号，自动生成，如 SO202605250001 |
| customer_id | bigint | 是 | 客户ID，关联客户表 |
| customer_name | varchar(100) | 是 | 客户名称（冗余） |
| contact_person | varchar(50) | 否 | 联系人 |
| contact_phone | varchar(20) | 否 | 联系电话 |
| receiver_name | varchar(50) | 是 | 收货人 |
| receiver_phone | varchar(20) | 是 | 收货电话 |
| receiver_address | varchar(500) | 是 | 收货地址 |
| total_amount | decimal(12,2) | 是 | 订单总金额（所有明细小计之和） |
| status | tinyint | 是 | 订单状态：0草稿/1待审核/2已审核/3已完成/4已取消 |
| remark | varchar(500) | 否 | 备注 |
| created_by | bigint | 是 | 创建人ID |
| created_at | datetime | 是 | 创建时间 |
| updated_at | datetime | 是 | 最后修改时间 |

### 5.2 实体：销售订单明细（sale_order_item）

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| id | bigint | 是 | 主键 |
| order_id | bigint | 是 | 关联销售订单ID |
| product_id | bigint | 是 | 商品ID，关联商品表 |
| product_name | varchar(200) | 是 | 商品名称（冗余） |
| price | decimal(12,2) | 是 | 单价 |
| quantity | int | 是 | 数量 |
| unit | varchar(20) | 是 | 单位 |
| subtotal | decimal(12,2) | 是 | 小计 = 单价 × 数量 |
