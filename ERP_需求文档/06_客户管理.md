# 客户管理模块

## 1. 模块概述
客户管理模块用于维护系统中所有客户的基础信息、联系方式、开票信息、收货地址等数据，同时管理客户的账户余额、欠款情况、交易记录等财务数据，为销售订单、财务核算提供客户基础数据支撑。

## 3. 业务逻辑

### 3.1 数据来源
| 数据 | 来源方式 | 来源说明 |
|------|---------|---------|
| 客户信息 | 用户手动创建 | 销售人员在页面上手动新建客户 |
| 账户余额 | 财务模块联动 | 客户预付款、退款自动更新；客户成单后核销预付款 |
| 欠款总额 | 销售模块联动 | 销售订单应收款、回款自动更新 |
| 尾单时间 | 销售模块联动 | 取最后一张销售订单的创建时间 |
| 成单金额 | 销售模块联动 | 销售订单金额 - 销售退货金额 |

### 3.2 状态定义
| 状态 | 含义 | 可执行操作 |
|------|------|-----------|
| 正常 | 正常状态，可被订单引用 | 编辑、转移、折扣设置 |
| 公共池 | 超过6个月未成单自动转入 | 查看、认领 |

### 3.3 状态流转
- 正常 → 公共池：超过6个月未成单自动转入
- 公共池 → 正常：业务员认领后转入

### 3.4 生命周期
1. 创建：销售人员新建客户信息，状态=正常
2. 使用：客户信息可被销售订单引用
3. 变更：根据业务需要可随时编辑客户信息
4. 休眠：超过6个月未成单自动转入公共池
5. 激活：业务员从公共池认领后恢复正常状态

### 3.5 跨模块联动
| 本模块操作 | 联动模块 | 联动效果 |
|-----------|---------|---------|
| 新增/修改客户信息 | 销售管理 | 新建订单时可选择最新客户信息 |
| 销售订单完成 | 客户管理 | 自动更新尾单时间、成单金额 |
| 销售退货完成 | 客户管理 | 自动扣减成单金额 |
| 财务收款 | 客户管理 | 自动扣减欠款总额 |
| 财务预付款 | 客户管理 | 客户成单后，核销预付款，自动更新减掉该订单金额之后的剩余款（账户余额 = 原预付款 - 订单金额） |

## 6. 权限管理

### 6.1 权限说明

| 功能 | 普通用户 | 管理员 |
|------|---------|--------|
| 查看客户列表 | ✓（只能看到自己的客户） | ✓（可以看到所有客户） |
| 创建客户 | ✓ | ✓ |
| 编辑客户 | ✓（只能编辑自己的客户） | ✓（可以编辑所有客户） |
| 删除客户 | ✓（只能删除自己的客户） | ✓（可以删除所有客户） |
| 客户转移 | ✓ | ✓ |
| 客户折扣设置 | ✓ | ✓ |

### 6.2 业务员可见性规则

**功能描述：**
- 普通用户只能看到并操作自己创建的客户（created_by = 当前用户ID）
- 管理员可以看到并操作所有客户（不受 created_by 限制）
- 业务员只能看到并且对状态为"正常"的客户执行操作（此规则对所有用户生效）

**重要规则**：业务员只能看到并且对状态为"正常"的客户执行操作。

**实现要点：**
- 前端：根据用户角色动态过滤可显示的客户列表
- 后端：查询时增加 created_by 过滤条件（仅对非管理员生效）
