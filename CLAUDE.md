# CLAUDE.md - AI 开发约束文档

> 本文档用于约束 AI（Claude/Trae 等）在处理本项目时的行为规范和输出标准。
> 任何 AI 助手在参与本项目开发前，必须先阅读并遵守本文档。

---

## 🎯 核心原则

### 1. 需求驱动
- **任何代码修改前，必须先阅读对应模块的 MD 需求文档**
- 不理解需求就写代码是严格禁止的
- 对需求有疑问时，先向用户确认，不要假设

### 2. 结构一致性
- 新增模块必须遵循现有文档结构和代码结构
- 不要引入新的目录结构或命名规范，除非用户明确要求
- 保持代码风格与现有代码一致

### 3. 简洁优先
- 单个函数复杂度不超过 20 行（不含注释和空行）
- 做好函数解耦，一个函数只做一件事
- 避免过度工程化，先实现核心功能

## 📝 文档规范

### 语言要求
- **所有输出必须使用中文**，包括：
  - 代码注释
  - 文档内容
  - 提交信息（commit message）
  - 变量命名（业务相关）
  - 用户交互提示

### 需求文档编写规范

当为用户编写或修改需求文档时：

1. **使用表格描述列表字段**
   ```markdown
   | 字段名 | 类型 | 说明 | 宽度建议 |
   |-------|------|------|---------|
   | 订单编号 | 文本 | 唯一标识 | 150px |
   ```

2. **状态流转用文字描述或代码块**
   ```markdown
   草稿 → 待审核：用户点击"提交审核"
   待审核 → 已审核：审批人点击"通过"
   ```

3. **跨模块联动用表格**
   ```markdown
   | 本模块操作 | 联动模块 | 联动效果 |
   |-----------|---------|---------|
   | 销售订单审核通过 | 库存管理 | 可生成出库单 |
   ```

4. **数据模型用表格**
   ```markdown
   | 字段名 | 类型 | 必填 | 说明 |
   |-------|------|------|------|
   | id | bigint | 是 | 主键 |
   ```

## 💻 前端开发规范

### 技术栈
- **HTML5** - 语义化标签
- **CSS3** - 原生 CSS，不使用预处理器
- **原生 JavaScript** - ES6+，不使用框架（Vue/React/Angular）
- **无构建工具** - 直接浏览器打开即可运行

### 代码风格

#### HTML
```html
<!-- 使用 4 空格缩进 -->
<!-- 属性按字母顺序排列 -->
<!-- 类名使用 kebab-case -->
<div class="page-header">
    <h2>销售订单</h2>
    <button class="btn btn-primary">新建</button>
</div>
```

#### CSS
```css
/* 使用 4 空格缩进 */
/* 选择器按层级分组 */
/* 属性按字母顺序排列 */
.btn {
    background: #1890ff;
    border: 1px solid #1890ff;
    border-radius: 4px;
    color: #fff;
    cursor: pointer;
    font-size: 13px;
    padding: 6px 16px;
}
```

#### JavaScript
```javascript
// 使用 4 空格缩进
// 函数使用小驼峰命名
// 常量使用全大写下划线
// 注释使用中文

/**
 * 显示指定模块
 * @param {string} moduleId - 模块ID
 */
function showModule(moduleId) {
    // 隐藏所有模块
    document.querySelectorAll('.module').forEach(function(m) {
        m.classList.add('hidden');
    });

    // 显示目标模块
    var target = document.getElementById(moduleId);
    if (target) {
        target.classList.remove('hidden');
    }
}
```

### 页面结构

每个模块的预览页面包含：

1. **列表页** - 数据列表 + 查询条件 + 操作按钮
2. **表单页** - 新建/编辑表单
3. **详情页** - 只读详情展示

## 🔧 开发流程

### 新增模块时的标准流程

1. **阅读现有文档**
   - 先阅读 `ERP_需求文档/01_销售管理.md` 了解文档结构
   - 再阅读 `ERP_前端预览/` 下的代码了解前端结构

2. **编写需求文档**
   - 复制 `01_销售管理.md` 的框架到目标模块
   - 按照业务需求填充内容
   - 保持表格格式一致

3. **开发前端预览**
   - 在 `index.html` 中添加新模块的导航
   - 添加对应的 section 内容
   - 确保样式与现有页面一致

4. **本地验证**
   - 启动本地服务器
   - 检查页面显示是否正常
   - 检查页面切换是否流畅

5. **提交代码**
   - 提交信息使用中文
   - 格式：`模块名：描述`
   - 示例：`采购管理：完成需求文档和前端预览`

## ❌ 禁止事项

1. **不要擅自修改项目结构**
   - 不要新建目录层级
   - 不要修改现有文件名
   - 不要引入新的技术栈

2. **不要过度设计**
   - 不要为简单功能写复杂代码
   - 不要引入设计模式除非必要
   - 不要写未在需求中提到的功能

3. **不要假设需求**
   - 不确定的地方先问用户
   - 不要替用户做决定
   - 不要省略需求中的细节

4. **不要使用英文注释**
   - 所有注释必须使用中文
   - 文档必须使用中文
   - 提交信息必须使用中文
