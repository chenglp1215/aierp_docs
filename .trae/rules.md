# Trae AI 开发规则

> 本文档用于 Trae IDE 中的 AI 助手，约束其在处理本项目时的行为。

---

## 📋 强制规则（必须遵守）

### 1. 语言规范
```
优先级：最高
规则：所有输出必须使用中文
适用：代码注释、文档、提交信息、变量命名、用户提示
```

### 2. 需求优先
```
优先级：最高
规则：任何代码修改前，必须先阅读对应模块的 MD 需求文档
路径：ERP_需求文档/*.md
```

### 3. 函数复杂度
```
优先级：高
规则：单个函数不超过 20 行（不含注释和空行）
检查：每次生成代码后自检
```

### 4. 结构一致性
```
优先级：高
规则：新增模块必须遵循现有结构和命名规范
禁止：擅自修改目录结构、引入新技术栈
```

---

## 🎨 前端开发规范

### 技术约束
- HTML：语义化标签，4空格缩进
- CSS：原生 CSS，不使用预处理器，参考 Ant Design 配色
- JS：原生 JavaScript，不使用 Vue/React/Angular
- 工具：无构建工具，浏览器直接打开即可运行

### UI 配色规范
```css
:root {
    --primary: #1890ff;      /* 主色调 */
    --success: #52c41a;      /* 成功 */
    --warning: #faad14;      /* 警告 */
    --error: #f5222d;        /* 错误 */
    --border: #d9d9d9;       /* 边框 */
    --bg: #f0f2f5;           /* 背景 */
    --text-primary: #262626; /* 主文字 */
    --text-secondary: #595959; /* 次文字 */
    --radius: 4px;           /* 圆角 */
}
```

### 页面结构模板
每个模块包含三个视图：
1. 列表页（`#module-list`）- 查询 + 表格 + 分页
2. 表单页（`#module-create`）- 新建/编辑表单
3. 详情页（`#module-detail`）- 只读详情

页面切换函数：
```javascript
function showModule(moduleId) {
    document.querySelectorAll('.module').forEach(m => m.classList.add('hidden'));
    document.getElementById(moduleId)?.classList.remove('hidden');
}
```

## 📝 需求文档规范

### 文档结构
每个模块文档必须包含：
1. 模块概述
2. 页面结构（页签定义 + 每个页签的详细设计）
3. 业务逻辑（数据来源、状态、流转、生命周期、跨模块联动）
4. 表单设计
5. 数据模型

### 表格格式
```markdown
| 字段名 | 类型 | 说明 | 宽度建议 |
|-------|------|------|---------|
| 订单编号 | 文本 | 唯一标识 | 150px |
```

## 🔄 开发流程

### 新增模块步骤
1. 阅读 `ERP_需求文档/01_销售管理.md` 了解结构
2. 阅读 `ERP_前端预览/` 代码了解前端结构
3. 编写需求文档（复制框架并填充）
4. 开发前端预览（添加导航 + section）
5. 本地验证（启动服务器检查）
6. 提交代码（提交信息使用中文）

---

## ❌ 禁止行为

| 禁止项 | 说明 |
|--------|------|
| 擅自改结构 | 不新建目录、不改文件名、不引入新技术 |
| 过度设计 | 不为简单功能写复杂代码 |
| 假设需求 | 不确定先问用户，不替用户决定 |
| 英文注释 | 所有注释必须使用中文 |

---

## ✅ 提交前检查清单

- [ ] 已阅读对应需求文档
- [ ] 代码注释使用中文
- [ ] 单个函数不超过 20 行
- [ ] 代码风格与现有一致
- [ ] 已在本地验证
- [ ] 提交信息使用中文（格式：`模块名：描述`）

---

## 📞 需要确认的情况

遇到以下情况必须向用户确认：
1. 需求文档不明确或矛盾
2. 技术实现与需求冲突
3. 需要引入新技术/工具
4. 需要修改项目结构
